# 넥서스 업로드용 파이썬 패키지 다운로드

Nexus PyPI 저장소에 업로드할 수 있도록 Python 패키지 파일을 Windows 환경에서 다운로드하고, 내부망에서 사용할 업로드 스크립트와 사용 방법을 함께 준비한다.

이 문서는 Windows PowerShell 실행을 기준으로 한다.

## 기본 원칙

- 사용자가 별도로 지시하지 않으면 현재 경로를 Python 프로젝트 폴더로 간주한다.
- 현재 경로에서 적절한 프로젝트 의존성 파일을 찾아 Nexus 업로드용 패키지를 다운로드할 수 있도록 진행한다.
- 의존성 기준을 판단하기 어렵거나 다운로드 범위가 애매할 때만 사용자에게 확인 질문을 한다.
- Nexus에 업로드할 파일은 wheel(`.whl`) 또는 source distribution(`.tar.gz`, `.zip`) 형식이어야 한다.
- 외부망 또는 작업 PC에서는 패키지 다운로드까지만 진행한다.
- Nexus 업로드는 내부망에서 독립적으로 진행하므로, 다운로드 결과물과 업로드 스크립트를 함께 전달할 수 있게 준비한다.

## 관련 파일

- 업로드 스크립트: `Upload-PypiPackagesToNexus.ps1`
- 내부망 업로드 사용법: `내부망에서-넥서스에-파이썬-패키지-업로드하는-방법.txt`

## 작업 순서

### 1. 현재 경로 기준으로 작업 시작

처음에는 현재 경로를 Python 프로젝트 폴더로 보고 작업을 시작한다.

사용자가 패키지 목록, requirements 파일, lock 파일, wheelhouse 경로 등을 별도로 지정한 경우에는 해당 지시를 우선한다.

현재 경로에서 기준을 정하기 어렵거나 여러 선택지가 충돌하면 사용자에게 아래처럼 질문한다.

```text
현재 경로에서 여러 의존성 기준이 확인되어 다운로드 기준을 정하기 어렵습니다.

아래 중 어떤 기준으로 진행할까요?
- 현재 프로젝트의 lock 또는 requirements 파일 기준
- 사용자가 제공하는 패키지 목록 기준
- 사용자가 지정하는 requirements 또는 lock 파일 경로 기준
- 이미 다운로드된 wheelhouse 폴더 기준

dev/test 의존성 포함 여부도 별도 기준이 있으면 알려주세요.
```

질문은 작업을 멈추기 위한 절차가 아니라, 자동 판단이 위험하거나 결과가 달라질 수 있을 때만 사용한다.

### 2. pip download로 패키지 다운로드

다운로드 작업에 사용할 Python 환경을 준비한 뒤, `pip download`로 업로드 대상 패키지를 `wheelhouse` 폴더에 내려받는다.

Python이 설치되어 있지 않으면 임의로 설치하지 말고, 사용자에게 설치 진행 여부와 사용할 Python 버전을 확인한다.

`pip` 업그레이드는 기본적으로 진행하지 않는다. 다운로드 과정에서 업그레이드가 필요하다고 판단될 때만 사용자에게 확인한 뒤 진행한다.

프로젝트에 `.venv`가 있으면 해당 가상환경을 사용할 수 있다. 없으면 새로 생성하지 말고 현재 사용 가능한 Python 환경으로 진행한다.

```powershell
New-Item -ItemType Directory -Force -Path wheelhouse
```

이미 활성화된 가상환경이 있거나 사용자가 별도 Python 경로를 지정한 경우에는 그 환경을 사용한다.

의존성 파일이 있으면 해당 파일 기준으로 다운로드한다.

```powershell
python -m pip download `
  --requirement requirements.txt `
  --dest wheelhouse
```

패키지 목록을 직접 받은 경우에는 패키지명을 지정해서 다운로드한다.

```powershell
python -m pip download `
  --dest wheelhouse `
  fastapi uvicorn pydantic
```

### 3. 다운로드 결과 검토

내부망 전달 전에 wheelhouse 폴더에 업로드 대상 파일이 있는지 확인한다.

```powershell
Get-ChildItem -LiteralPath wheelhouse -File |
  Where-Object { $_.Extension -in ".whl", ".zip", ".gz" }
```

사용자에게 다운로드된 파일 개수와 주요 패키지명을 요약한다.

```text
다운로드 결과:
- wheelhouse 경로: C:\work\project\wheelhouse
- 다운로드 파일 수: 32개
- 주요 패키지: fastapi, uvicorn, pydantic

내부망으로 가져갈 항목:
- wheelhouse 폴더 전체
- Upload-PypiPackagesToNexus.ps1
- 내부망에서-넥서스에-파이썬-패키지-업로드하는-방법.txt
```
